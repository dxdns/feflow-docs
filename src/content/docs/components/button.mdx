A versatile button component with support for visual variants and states like `disabled`.

## Usage

<Tabs>
  <TabItem label="Demo">
    <Button disabled>Disabled</Button>
	<br />
	<br />
	<Button variant="text">Text</Button>
	<br />
	<br />
	<Button variant="outlined">Outlined</Button>
	<br />
	<br />
	<Button variant="contained">Contained</Button>
  </TabItem>
  <TabItem label="Code">
    ```svelte
	<script lang="ts">
		import { Button } from "@dxdns/svelte-components"
	</script>

	<Button disabled>Disabled</Button>
	<br />
	<br />
	<Button variant="text">Text</Button>
	<br />
	<br />
	<Button variant="outlined">Outlined</Button>
	<br />
	<br />
	<Button variant="contained">Contained</Button>
	```
  </TabItem>
</Tabs>

<br />
<br />

| Prop       | Type                                      | Default       | Description                                     |
| ---------- | ----------------------------------------- | ------------- | ----------------------------------------------- |
| `variant`  | `"text"` \| `"outlined"` \| `"contained"` | `"contained"` | Sets the visual style of the button.            |
| `disabled` | `boolean`                                 | `false`       | Disables the button, making it non-interactive. |
| `type`     | `"button"` \| `"submit"` \| `"reset"`     | `"button"`    | Defines the button type, useful in forms.       |

| Slot     | Description                                      |
|----------|--------------------------------------------------|
| default  | Content rendered inside the button (text, icons) |

# Accessibility
This component uses the native `<button>` element and supports standard HTML attributes like `aria-*`.

# Best Practices
- Use `variant="text"` for low-emphasis actions.
- Use `variant="contained"` for primary calls to action.
- Combine icons and text for better context when needed.
